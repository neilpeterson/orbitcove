## Design Principles

### 1. Grandma-Friendly
If a 70-year-old grandparent can't figure out how to RSVP to a family dinner, we've failed. Every core action should be achievable in 2-3 taps with clear, jargon-free labels.

### 2. Community Context is Always Clear
Users may be in multiple communities. The current community should be visually prominent at all times. Switching communities should be fast but intentional (no accidental posts to wrong group).

### 3. Progressive Disclosure
Power features exist but don't clutter the primary experience. Recurring events, custom splits, and advanced settings are available but tucked away.

### 4. Offline Confidence
Users should never wonder "did that work?" Actions taken offline should feel instant with clear sync status indicators.

### 5. Consistent Patterns
Same gestures and interactions across all pillars:
- Swipe to reveal actions
- Pull to refresh
- Long press for quick actions
- Bottom sheets for creation flows

## Accessibility Considerations

### VoiceOver Support

All interactive elements must have:
- Descriptive accessibility labels
- Proper accessibility traits
- Logical focus order

**Example**:
```swift
Button("Going") { ... }
    .accessibilityLabel("RSVP Going to Practice on January 25")
    .accessibilityHint("Double tap to mark yourself as going")
```

### Dynamic Type

- All text must scale with Dynamic Type
- Minimum touch target: 44x44 points
- Layouts must adapt to larger text sizes
- Test at accessibility sizes (AX1-AX5)

### Color & Contrast

- Minimum contrast ratio: 4.5:1 for text
- Never use color alone to convey information
- Support Increase Contrast setting
- Support Reduce Motion setting (disable animations)

### Interaction Patterns

- All swipe actions must have alternative (tap into detail â†’ action button)
- Long-press menus must have visible menu button alternative
- Support full keyboard navigation (for external keyboards)
